# 提示工程概述

提示工程（Prompt Engineering）是设计和优化大语言模型输入提示的技术，以引导模型产生更准确、更有用的输出。

## 为什么提示工程很重要？

在 LLM 时代，**提示就是编程**。同样的模型，不同的提示可以产生截然不同的结果：

- 糟糕的提示 → 模糊、错误、不相关的回答
- 精心设计的提示 → 准确、结构化、高质量的输出

## 基本原则

### 1. 清晰明确

❌ "告诉我关于 AI 的事"

✅ "用 3-5 个要点，解释 2024 年人工智能领域最重要的 3 个技术突破，每个突破包含：名称、原理简述、实际应用案例"

### 2. 提供上下文

告诉模型它的角色、目标受众和输出格式。

### 3. 给出示例

通过示例展示期望的输入-输出模式，比纯文字描述更有效。

### 4. 逐步推理

对于复杂任务，引导模型分步思考而非直接给出答案。

## 提示 vs 微调

| 维度 | 提示工程 | 微调 |
|------|---------|------|
| 成本 | 低（仅推理） | 高（需要训练） |
| 灵活性 | 高（随时修改） | 低（需重新训练） |
| 效果上限 | 中等 | 高 |
| 知识更新 | 实时 | 需重新训练 |
| 适用场景 | 快速原型、通用任务 | 专用领域、极致性能 |
