# 基础技巧

掌握这些提示工程基础技巧，就能显著提升大语言模型的输出质量。

## Zero-shot 提示

直接给出指令，不提供示例：

```
将以下文本分类为正面、中性或负面情感：

文本：这款产品的用户体验非常流畅，但价格偏高。
情感：
```

## Few-shot 提示

提供少量示例帮助模型理解任务模式：

```
将评论分类为正面或负面：

评论：服务态度很好 → 正面
评论：等待时间太长了 → 负面
评论：菜品味道不错但环境一般 →
```

## 角色扮演

赋予模型一个专业角色，让输出更专业：

```
你是一位资深的机器学习工程师，擅长将复杂概念解释给初学者。
请解释什么是梯度下降算法，使用生活中的类比。
```

## 输出格式控制

明确指定输出格式：

```
请分析以下代码的时间复杂度，用以下 JSON 格式输出：
{
  "complexity": "O(?)",
  "explanation": "一句话解释",
  "bottleneck": "瓶颈所在行"
}
```

## 约束与否定

告诉模型什么**不要**做，和告诉它做什么一样重要：

```
用 200 字以内解释量子计算，要求：
- 不要使用数学公式
- 不要使用"简单来说"这类填充词
- 面向高中生水平
```

## 思维链触发

通过添加"让我们一步步思考"来触发推理模式：

```
一个商店的苹果价格是橙子的 2 倍，买了 3 个苹果和 5 个橙子共花 44 元，橙子多少钱？

让我们一步步思考。
```

## 迭代优化

提示工程是迭代过程：

1. 编写初始提示
2. 评估输出
3. 分析不足
4. 修改提示
5. 重复直到满意
